### ⚠️ Список нюансов для возможной передачи генератору

1. **Строка 372** — «для префиксных `LIKE 'foo%'` … если использовать operator class `text_pattern_ops`». Официальная документация PostgreSQL уточняет: специальный operator class (`text_pattern_ops`) нужен **только если БД работает не в локали C**; в локали C префиксный `LIKE` индексируется и стандартным `text_ops`. Текущая формулировка подразумевает, что `text_pattern_ops` нужен всегда. Источник: <https://www.postgresql.org/docs/current/indexes-types.html>
2. **Строка 131 / 200** — «Все листовые узлы связаны в двусвязный список». Корректно и для канонического B+-дерева, и для PostgreSQL. Уточнение: в PostgreSQL (алгоритм Lehman & Yao, «Btree+») в двусвязный список связаны страницы **каждого уровня**, а не только листового. Для листьев утверждение полностью верно. Источник: <https://www.postgresql.org/docs/current/btree-implementation.html>
3. **Строка 386** — Ramakrishnan & Gehrke, 3rd ed. указан год **2003**. Это год copyright; фактическая дата издания McGraw-Hill — август 2002.
